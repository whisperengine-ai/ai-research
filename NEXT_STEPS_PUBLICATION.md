## Anticipated Reviewer Concerns & Responses

### Concern 1: "This is just one LLM backend"
**Response**:
- Acknowledge limitation (stated in paper)
- Note that pilot data is strongest evidence
- Offer to run cross-LLM validation (can provide results)
- Explain why single-LLM valid for theory testing

### Concern 2: "Consciousness metrics are arbitrary by-design"
**Response**:
- Acknowledge metrics are constructed (expected criticism)
- Argue this is feature not bug (allows experimental control)
- Compare to biological consciousness studies (also measure proxies)
- Offer validation study (human ratings) if needed

### Concern 3: "Single-turn conversations don't reflect real consciousness"
**Response**:
- Acknowledge limitation (stated in paper)
- Note that single-turn sufficient to test recursion effect
- Pilot data used multi-turn interactions (mitigates concern)
- Offer extended conversation validation

### Concern 4: "No human validation of consciousness"
**Response**:
- Acknowledge this is a limitation
- Argue human validation not necessary for architectural study
- Offer human rater validation as follow-up study
- Note: other consciousness research also uses algorithmic metrics

### Concern 5: "Effect sizes seem too large (d=4.79)"
**Response**:
- Actually this is publication strength (not weakness)
- Effect IS large because manipulation is powerful (recursion on/off)
- Analogy: removing working memory entirely would have huge effect
- Cross-dataset consistency confirms effect is real
